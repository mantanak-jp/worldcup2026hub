# Product Requirements

この文書は、WorldCup2026Hub の完成時に満たすべきプロダクト要求を定義する。

`docs/site_concept.md` が「なぜこのサイトを作るか」「どのような価値を提供するか」を説明するのに対し、この文書は「利用者が何をできるか」「システムが何を満たす必要があるか」「何を対象外とするか」を定義する。

## 1. Product Purpose

WorldCup2026Hub は、2026 FIFA World Cup の各試合について、日本語を含む複数言語・複数地域の戦術レビュー、試合分析、現地メディア記事、公式情報、監督・選手コメント、統計情報などを参照し、それらを整理・照合した独自の日本語レビューを提供するサイトである。

利用者は各国のサイトを個別に巡回しなくても、主要な論点を日本語で確認できる。より詳しく確認したい場合は、明示されたソースから原典へ移動できる。

このサイトは、外部記事の転載サイト、記事全文の翻訳サイト、長文引用の保存場所、単純なリンク集ではない。

## 2. Target Users

主な対象利用者は、次のような人である。

- ワールドカップの試合を結果だけでなく戦術面から理解したい人
- 日本語以外の良質な分析も参照したいが、各国語の記事を自力で探して読み続ける時間がない人
- 日本代表、特定の代表チーム、監督、選手、大会全体の戦術的変化を継続して追いたい人
- レビューの結論だけでなく、その根拠となるソースや情報量も確認したい人

## 3. User Problems

WorldCup2026Hub は、次の課題を解決する。

- 戦術レビューが言語・国・媒体ごとに分散している
- 検索、SNS、ニュースサイト、戦術サイトを個別に巡回する負担が大きい
- 日本語だけでは拾えない視点がある
- 単一の記事だけでは、論点の偏りや評価の違いを判断しにくい
- 自動生成レビューが、どの程度の根拠に基づくのか分かりにくい
- 情報不足の試合でも、もっともらしい記事が生成される危険がある

## 4. Core User Value

利用者に提供する中心的価値は次のとおりである。

1. 各国サイトを毎回巡回せず、日本語で試合レビューを読める
2. 複数ソースの共通点と相違点を一つのレビューで確認できる
3. 通常は WorldCup2026Hub だけで概要を理解できる
4. 必要なときだけ原典へ移動して確認できる
5. レビューの source coverage、confidence、status、missing inputs を見て、情報の充実度を判断できる
6. 試合単位・チーム単位で大会の進行と戦術的変化を追える

## 5. Primary User Journeys

### 5.1 試合から読む

利用者は試合一覧から対象試合を選び、次の内容を確認できる。

- 試合結果と基本情報
- 試合展開の要約
- 初期配置と主要な戦術テーマ
- 両チームの攻撃・守備の特徴
- 試合中の修正、交代、転換点
- 主要選手のパフォーマンス
- ソース間で一致した評価
- ソース間で異なる評価
- 参照ソース一覧
- source coverage、confidence、generation version、generated time、review status、missing inputs

### 5.2 チームから追う

利用者はチーム一覧から対象チームを選び、関連試合とレビューをたどれる。

完成時には、同一チームについてグループステージから決勝トーナメントまでの変化や、複数試合を通じた戦術傾向を追えることを目指す。

### 5.3 原典を確認する

利用者はレビューを通常どおり日本語で読んだうえで、必要に応じて次を確認できる。

- ソース名
- ソース種別
- 言語
- 対象記事
- 元記事へのリンク
- 各論点を支える source ID / article ID

## 6. Functional Requirements

### FR-1 試合一覧

- 全試合を同一構造で表示できる
- 特定チームだけを前提にした構造にしない
- 試合状態、結果、レビュー状態を確認できる
- 試合詳細ページへ移動できる

### FR-2 チーム一覧

- 全チームを同一構造で表示できる
- チーム詳細ページへ移動できる
- 関連試合を確認できる

### FR-3 試合詳細

- 基本情報、結果レポート、戦術レビュー、生成レビュー、参照ソース、更新履歴を表示できる
- 情報が存在しない場合は、未取得・未生成・不足を明示する

### FR-4 チーム詳細

- チーム基本情報、関連試合、関連レビューを表示できる
- 将来的に大会中の戦術傾向や変化を追加できる構造を持つ

### FR-5 ソース表示

- レビューで参照したソースを明示する
- ソースから元記事へ移動できる
- ソース種別・言語・確認状態を表示できる

### FR-6 品質表示

各生成レビューには、少なくとも次を表示する。

- source coverage
- confidence
- generation version
- generated time
- review status
- source list and categories
- missing inputs

### FR-7 更新履歴

- レビューの生成・更新履歴を追跡できる
- 新しいソースが追加された場合、レビューを更新できる

## 7. Review Content Requirements

独自日本語レビューは、利用可能な根拠に応じて次の内容を扱う。

- 試合展開
- 初期配置
- 主な攻撃・守備構造
- プレス、保持、トランジション、セットプレーなどの主要テーマ
- 試合中の修正
- 交代の影響
- 転換点
- 主要選手
- 複数ソースの共通認識
- ソース間の見解の相違

レビュー本文は単一記事の翻訳や言い換えではなく、複数ソースを整理・照合して構成した独自の日本語文章とする。

## 8. Source Transparency Requirements

- 各 tactical claim は、可能な限り supporting source IDs と article IDs に紐づける
- 意見が分かれる論点では、対立するソースを記録する
- 参照したソース数、記事数、言語、カテゴリを表示する
- 利用者が元記事を確認できるリンクを提供する
- 出典不明の主張を公開レビューに含めない

## 9. Quality and Confidence Requirements

- confidence は正確性の保証ではなく、利用可能な構造化ソースと抽出情報に基づく品質シグナルとして扱う
- source coverage が低い場合は明示する
- missing inputs を隠さない
- low confidence や insufficient sources のレビューを表示する場合は、その状態を明確に示す
- 情報不足時に、長さを埋めるための推測や一般論を追加しない
- 十分な根拠がない場合は、短いレビュー、未生成、またはソース不足として扱う

## 10. Automation Requirements

完成時の通常運用は次を目標とする。

- 承認済みソースの自動収集
- 記事メタデータと短い抽出ノートの構造化
- 記事と試合・チームの関連付け
- 言語とソース種別の分類
- tactical claims と review outlines の生成
- 独自日本語レビューの生成
- 静的ページの更新と公開
- 新しい記事が見つかった場合の再生成・更新

大会運用時に、すべてのレビューについて人手承認を必須とはしない。ただし、自動公開は承認済みソース、承認済みworkflow、品質表示、失敗時の安全な停止を前提とする。

## 11. Copyright and Source Guardrails

次を必須とする。

- robots.txt、利用規約、著作権、取得方法をソースごとに確認する
- 承認済みソースだけを実収集対象にする
- 記事本文を原則として保存しない
- 記事全文を転載しない
- 記事全文を翻訳して掲載しない
- 長い引用を掲載しない
- 外部画像を無断保存・転載しない
- 原文に近すぎる要約を作成しない
- ソースポリシーで許可されないコンテンツを保持しない

## 12. Non-Goals

WorldCup2026Hub は、完成時にも次を目的としない。

- 外部記事の転載・翻訳まとめサイト
- 記事本文や外部画像のアーカイブ
- 単純なニュースリンク集
- 独自の試合映像解析による、ソースに存在しない戦術発見
- ソース根拠なしの監督意図の推定
- 情報不足を隠した、もっともらしい長文生成
- 有料API、秘密情報、課金アカウントへの必須依存
- ユーザーログイン、コメント、投稿、SNS機能
- 動画配信や試合映像の提供

戦術レビューの目標範囲は Level 3: source-based structured tactical review とする。Level 4 の独自映像解析や、人間の専門記者と同等の一次分析は、この要求仕様の対象外である。

## 13. Non-Functional Requirements

- 利用者向けUIとレビュー本文の基本言語は日本語とする
- 技術docs、内部field名、source metadata は必要に応じて英語を維持できる
- スマートフォンで主要ページを読める
- JavaScript無効時の完全対応は現時点の必須要件としない
- 公開ページは、サンプル・dry-run・実データの違いを誤認させない
- 失敗時は古い正常データを壊さず、状態を明示する
- コード変更はPRベースで管理する

## 14. Completion Criteria

プロダクト完成の最低条件は次のとおりである。

- 全試合・全チームの基本ページが存在する
- 承認済みソースを使った実収集が動作する
- 記事メタデータと抽出情報が試合・チームに紐づく
- 各試合について、情報量に応じた独自日本語レビューを生成できる
- ソース一覧と原典リンクを表示できる
- source coverage、confidence、status、missing inputs を表示できる
- 情報不足時に無理なレビュー生成を行わない
- 自動更新と公開が安全に動作する
- 転載、全文翻訳、長文引用、無断画像保存を行わない
- スマートフォンで主要な利用導線を確認できる
